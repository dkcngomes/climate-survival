## 2026-06-29 Climate Survival — Open-Meteo Seasonal API Fix

### Fixed: Climate forecast returning all zeros
**Problem:** The `FetchSeasonalAnomaliesAsync` method in `ClimateService.cs` used outdated Open-Meteo Seasonal API variable names:
- `temperature_2m_mean_anomaly` (no longer exists as daily variable)
- `precipitation_sum_anomaly` (no longer exists as daily variable)
- `extreme_forecast_index_temperature_2m` (no longer exists)
- `extreme_forecast_index_precipitation_sum` (no longer exists)

**Root cause:** Open-Meteo changed their Seasonal Forecast API. The anomaly and EFI variables moved from daily to weekly namespace.

**Fix applied:**
- Changed URL to use `weekly=temperature_2m_anomaly,precipitation_anomaly,temperature_2m_efi,precipitation_efi`
- Added `daily=temperature_2m_mean,precipitation_sum` for future chart data
- Updated `SeasonalApiResponse` DTO to parse `weekly` (not daily) for anomaly data
- Added `SeasonalWeekly` model with correct JSON property names

**Verified (local + live):**
- Colombo: tempAnomaly=0.53°C, precipAnomaly=-1.46mm, efiTemp=0.4
- Sigiriya: tempAnomaly=0.66°C
- London: tempAnomaly=3.8°C
